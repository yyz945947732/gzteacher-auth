### 为什么使用

- #### 完善的单元测试
  权限组件与工具函数都编写了丰富的测试用例。
- #### 提供强大的二次封装能力

  如果目前的权限组件无法满足你的需求，或者你希望你的权限组件是基于其他组件库，这些场景你都可以基于基础权限组件 `Wrapper` 等或工具方法 `useAuthData` 来轻易的封装你的权限组件。

  例如你想封装一个基于 `antd` 的权限表格组件，你可以：

  ```typescript
  import { Table } from "antd";
  import { useAuthData } from "@gzteacher/auth";

  function AuthTable(props) {
    const { columns, ...otherProps } = props;
    const authedColumns = useAuthData(columns); // 对数据驱动的字段使用 useAuthData 过滤权限
    return <Table columns={authedColumns} {...otherProps} />;
  }
  ```

  这样便实现了一个基本的能过滤权限数据的表格组件。

- #### 提供强大的本地调试能力

  你可以基于[验证器](/docs/stories-验证器--docs)轻易的对本地权限进行调试，包括但不限于：

  - 根据开发环境/正式环境动态分配权限。
  - 直接开启/关闭所有权限。
  - 直接开启/关闭某个模块所有权限。
  - 直接开启/关闭某个权限。

- #### 提供更灵活的配置能力

  你可以基于[验证器](/docs/stories-验证器--docs)更灵活的添加你自己想要的权限功能，包括但不限于：

  - 更灵活的权限代理。
  - 拥有某个权限则拥有所有权限。
  - 拥有某个权限则拥有某个模块所有权限。
  - 拥有某个权限则不拥有所有权限。

- #### 更细粒度的权限控制

  你可以在想要的任何地方添加任意数量的 `Provider`，这在不同模块需要请求不同权限接口或是权限由前端写死的情况下会很有用。
